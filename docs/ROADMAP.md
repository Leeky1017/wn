# WriteNow 开发路线图（建议）

## V1（0–6 个月）：可用的核心闭环

目标：**写作 → AI 改写 → diff 审核 → 版本回退 → 平台导出** 形成稳定习惯。

- 文档工作区：文件树、打开/新建、快捷键（`Ctrl/Cmd+K`、`Ctrl/Cmd+S`）
- AI 修改系统：选区指令、流式进度、unified diff 预览、应用后写入
- 轻量版本管理：自动快照、列表回退、标注 actor/reason
- 排版导出：公众号/知乎/小红书模式（HTML + 模板 CSS），导出下载
- 最小可观测：日志与错误可定位（WebSocket/HTTP）

验收指标：
- 关键链路“选区→指令→diff→应用”稳定，错误率可控
- 快照回退可靠，不丢稿
- 排版预览能覆盖主流结构（标题/列表/引用/代码块）

## V2（6–12 个月）：创作流程与平台闭环

目标：从“写作工具”变成“创作流程工具”。

- 内容组织 Agent：多层级大纲、章节依赖、结构重排（多步骤工具调用）
- 素材库：PDF/网页/图片导入，抽取要点与可检索引用
- 发布：公众号草稿箱直连（优先），知乎/小红书（RPA 或 API）
- 合规检查：违禁词/敏感词/段落长度/标题层级等规则体系
- 可选云同步：多端同步 + 冲突解决（V2 可选，不强制）

## V3（12–24 个月）：规模化与数据壁垒

目标：形成“创作者如何用 AI 写作”的场景数据与可复用工作流。

- 工作流编排：可复用的 Agent 流程（选题→大纲→初稿→润色→排版→发布）
- 平台效果追踪：阅读/收藏/转化数据回流，驱动改写建议
- 个性化写作风格：风格样本库、偏好学习、同一作者一致性
- 多 Agent 协作：结构编辑/事实核验/标题生成/SEO 优化等子 Agent
